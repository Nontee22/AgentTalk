# 项目优化方向

基于当前项目完成状态的全面评估，规划未来迭代方向。

---

## 一、功能增强

### 1. 多角色对话 / 群聊模式

当前一个对话只有一个角色，无法实现角色之间的互动。

- 扩展为「场景」模式：一个世界观下多角色同时参与对话
- `Conversation` 模型增加 `character_ids: list[UUID]` 多角色关联
- `prompt_builder` 支持多角色身份注入，轮流或并行回复
- 前端消息列表区分不同角色的头像和样式

**预计工作量**：2-3 天

### 2. 故事线 / 剧情分支

当前对话是线性的自由聊天，缺少叙事引导。

- 引入 `StoryNode` 模型：预设关键剧情节点 + 触发条件
- 对话到达某阶段自动推进剧情（LLM 判断触发条件是否满足）
- 支持分支选择，不同选项导向不同剧情线
- 适合打造"互动小说"体验

**预计工作量**：3-5 天

### 2. 角色情感状态系统

当前角色无状态，每轮对话性格不变。

- 新增 `CharacterState` 模型：`mood`（情绪）、`affinity`（好感度）等动态属性
- 情感状态随对话内容由 LLM 自动更新
- 情感写入 system prompt，让同一角色在不同对话中产生差异化回复
- 前端角色面板展示当前情感状态

**预计工作量**：1-2 天
### 4. 记忆系统增强

当前记忆仅提取事实/关系/偏好/事件四类。

- 增加**情感记忆**类别（用户让角色开心/生气的事件）
- 支持**跨角色记忆共享**：同一世界观下不同角色共享世界级事件记忆
- 前端增加记忆编辑功能（目前只能查看和删除）
- 8s 延迟刷新改为后端 SSE 事件推送通知

**预计工作量**：1-2 天

## 二、技术架构

### 5. 测试体系

当前零测试覆盖，是最大的技术债。

**后端测试**（优先）：
- `chat_service`：流式逻辑、消息存储、记忆触发
- `memory_service`：提取准确性、去重逻辑、召回排序
- `prompt_builder`：上下文窗口裁剪、token 预算
- API 集成测试：权限校验、分页、错误响应

**前端测试**：
- `chatStore`：发送/重试/编辑/停止等状态流转
- `ChatMessage` 组件：编辑模式、重新生成按钮条件渲染

**工具**：pytest + pytest-asyncio（后端），Vitest（前端）

**预计工作量**：2-3 天
