# 项目优化清单

基于前后端全面审计，按优先级分类整理。

---

## P0：必须修复（用户直接受影响）

### 前端

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| F1 | 世界书详情页无错误状态 | `WorldDetailView.vue` | fetch 失败时白屏，无任何提示 |
| F2 | 表单提交错误被静默吞掉 | `WorldFormView.vue`, `CharacterFormView.vue` | 无 catch 块，保存失败无反馈 |
| F3 | 无 404 路由 | `router/index.ts` | 无效 URL 显示空白页 |
| F4 | 无路由滚动行为 | `router/index.ts` | 页面切换后保持旧滚动位置 |
| F5 | 图片上传无文件大小校验 | `ImageUpload.vue` | 界面写"最大5MB"但未实际校验 |
| F6 | SSE 不走 axios 拦截器 | `api/chat.ts` | token 过期直接跳登录，不尝试刷新 |

### 后端

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| B1 | JWT 默认密钥不安全 | `core/config.py` | 生产环境如果没改 `.env` 则使用已知密钥 |
| B2 | 文件上传无内容校验 | `api/upload.py` | 只检查扩展名，不校验实际 MIME 类型 |
| B3 | 登录接口无限流 | `api/auth.py` | 可暴力破解密码 |
| B4 | Refresh Token 无失效机制 | `api/auth.py` | 被盗的 token 7天内持续有效 |
| B5 | CORS 写死 localhost | `main.py` | 部署时需改代码 |
| B6 | start_conversation 不校验角色-世界归属 | `chat_service.py` | 可创建角色A+世界B的混搭对话 |

## P1：体验提升（用户会感受到差异）

### 前端

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| F7 | 世界书列表无分页 | `WorldListView.vue` | 世界书多了一次全部加载 |
| F8 | 对话消息无分页/无限滚动 | `ChatView.vue` | 固定取50条，旧消息无法查看 |
| F9 | FoldableSection 的 defaultOpen 未生效 | `FoldableSection.vue` | 声明了 prop 但没用 |
| F10 | 标签筛选写死4个 | `WorldListView.vue` | 新标签不会出现在筛选栏 |
| F11 | 表单离开无未保存提醒 | 表单页 | 编辑中意外离开丢失内容 |
| F12 | 角色抽屉打开无加载提示 | `WorldDetailView.vue` | 点击后 fetch 期间无反馈 |
| F13 | 无页面切换过渡动画 | `App.vue` | 页面切换生硬 |
| F14 | 流式光标动画太慢 | `ChatView.vue` | `animate-pulse` 2秒一次，不像打字光标 |
| F15 | 无 `<meta name="color-scheme" content="dark">` | `index.html` | 加载时可能闪白屏 |
| F16 | icon-only 按钮缺少 aria-label | 全局 | 无障碍访问缺失 |

### 后端

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| B7 | 对话列表无分页 | `api/chat.py` | 返回用户所有对话，无 LIMIT |
| B8 | 角色列表无分页 | `api/characters.py` | 一个世界下所有角色无分页 |
| B9 | 全部消息加载到内存 | `chat_service.py` | 只用最近20条但全部 SELECT |
| B10 | 文本字段无长度限制 | `schemas/` | 世界书设定/角色背景可提交 MB 级文本 |
| B11 | max_context_tokens 写死 6000 | `prompt_builder.py` | 32K/64K 模型浪费大量上下文 |
| B12 | N+1 查询 | `world_service.py` | 每个世界书单独查角色数 |
| B13 | 缺少 message (conversation_id, created_at) 复合索引 | `models/message.py` | 对话消息排序慢 |
| B14 | JSONB tags 缺少 GIN 索引 | `models/world.py` | 标签过滤查询无法用索引 |
| B15 | 权限检查逻辑重复 | `api/characters.py`, `api/worlds.py` | 6个端点复制粘贴同样的 if/else |
| B16 | deprecated asyncio.get_event_loop() | `embedding_service.py` | Python 3.12 已弃用 |

## P2：锦上添花（细节打磨）

### 前端

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| F17 | 记忆刷新用 magic 8秒延迟 | `chatStore.ts` | 后端快慢都不适配 |
| F18 | Header 下拉用 setTimeout 150ms 关闭 | `AppHeader.vue` | 在移动端/触屏上有闪烁 |
| F19 | authStore 有未使用的 useRouter import | `authStore.ts` | 死代码 |
| F20 | 路由守卫直接读 localStorage 而非 store | `router/index.ts` | 和 store 状态可能不同步 |
| F21 | textarea 无字数提示 | 表单页 | 用户不知道有多长限制 |
| F22 | retryLastMessage 会丢失原始消息 | `chatStore.ts` | 重试失败后 UI 上看不到原消息 |

### 后端

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| B17 | stream_registry 是进程级内存 | `stream_registry.py` | 多 worker 时取消流式可能失败 |
| B18 | Redis 已连接但未使用 | `database.py` | 计划中的缓存功能未实现 |
| B19 | health 接口降级时仍返回 200 | `api/health.py` | 负载均衡器无法检测异常 |
| B20 | token 估算粗糙 | `token_counter.py` | 中英混排误差大 |
| B21 | 模型无 __repr__ | 所有 model | 调试日志不友好 |
| B22 | embedding 模型加载在 web 进程 | `embedding_service.py` | 占内存且阻塞首次请求 |

## 建议实施顺序

**第一批（快速修复，1-2小时）**：
- F1-F5（前端错误处理 + 404 路由 + 滚动行为）
- B5（CORS 改为环境变量）
- B6（角色-世界归属校验）

**第二批（安全加固，半天）**：
- B1（JWT 密钥检查）
- B2（上传文件校验）
- B3（登录限流）
- B10（字段长度限制）

**第三批（性能优化，半天）**：
- B9（消息查询加 LIMIT）
- B12（N+1 修复）
- B13-B14（索引）
- F7-F8（分页）

**第四批（Phase 5 剩余功能）**：
- F13（页面过渡动画）
- F10（动态标签）
- Phase 5.2-5.4（角色推荐、情绪、角色切换）
