# 角色扮演对话系统 — 开发计划

## 一、产品核心概念

```
世界书 (World Book)
├── 世界观设定（历史、规则、地理、势力、魔法体系等）
├── 角色 A（性格、经历、关系、说话风格）
├── 角色 B
├── 角色 C
└── ...

用户选择一个世界书 → 选择其中的角色 → 开始对话
对话必须同时遵守：世界书的背景规则 + 角色的个性设定
```

---

## 二、技术选型

| 层 | 技术 | 职责 |
|----|------|------|
| 前端 | Vue 3 + TypeScript + Tailwind CSS | 聊天 UI、世界书/角色浏览 |
| 后端 | Python 3.12 + FastAPI | REST API、SSE 流式推送、业务逻辑 |
| 数据库 | PostgreSQL 16 | 世界书、角色、用户、会话、消息 |
| 缓存 | Redis 7 | 会话缓存、热门数据缓存、限流 |
| LLM | 国内模型 API（DeepSeek / 通义千问 / ChatGLM） | 通过 OpenAI 兼容接口调用 |
| 向量库 | PGVector（PostgreSQL 扩展） | 长期记忆语义检索（后期） |

## 三、前端设计方案

### 设计原则

- **PC 端优先**：所有页面以 1280px+ 桌面浏览器为主要适配目标
- **沉浸感**：暗色系基调，让用户有"进入另一个世界"的感觉
- **克制**：不堆砌特效，用排版、色彩、留白营造氛围

### 视觉风格

| 项目 | 方案 |
|------|------|
| 基调 | 深色主题（近黑底 `#0f0f14`），辅以暖金色点缀 `#c9a855` |
| 字体 | 标题/角色名用衬线体（思源宋体），正文/对话用无衬线体（思源黑体）|
| 卡片 | 圆角 12px，微透明背景 `rgba(255,255,255,0.04)`，细边框 `rgba(255,255,255,0.08)` |
| 交互 | 悬停时卡片微微上浮 + 边框变亮，过渡 300ms ease |
| 间距 | 统一 8px 网格系统 |
| 图标 | Lucide Icons（线性风格，与暗色主题搭配） |

### 配色系统

```
背景层：
  --bg-deep:    #0a0a0f     ← 最底层
  --bg-base:    #0f0f14     ← 页面底色
  --bg-surface: #16161d     ← 卡片/面板
  --bg-hover:   #1e1e28     ← 悬停态

文字层：
  --text-primary:   #e8e6e3  ← 正文
  --text-secondary: #8b8994  ← 次要信息
  --text-muted:     #4a4853  ← 禁用/提示

强调色：
  --accent:         #c9a855  ← 金色，用于标题、重要按钮、选中态
  --accent-hover:   #dbb960  ← 金色悬停
  --accent-dim:     #2a2418  ← 金色背景（极淡）

功能色：
  --success:  #4ade80
  --danger:   #f87171
  --info:     #60a5fa
```

**三栏比例**：左侧 240px 固定 / 中间自适应 / 右侧 300px 固定（可折叠）

**左侧 — 会话列表栏**：
- 顶部搜索框，下方为会话列表
- 每条显示：角色头像缩略图、角色名、最后一条消息摘要、时间
- 当前会话高亮（左边框金色条）
- 底部「+ 新对话」按钮

- 角色消息左对齐，用户消息右对齐
- 角色消息带角色名标签 + 小头像
- 流式输出时末尾显示闪烁光标 `█`
- 支持 Markdown 渲染（代码块、列表、加粗等）
- 底部输入框：圆角，右侧发送按钮，支持 Shift+Enter 换行
- 新对话时自动展示角色的 greeting 作为第一条消息

**右侧 — 角色/世界信息栏**：
- 角色头像 + 名字 + 身份
- 折叠式展示：性格、经历、关系（默认折叠，避免信息过载）
- 底部简要世界背景 + 「查看完整世界观」链接
- 「切换角色」按钮（在同一世界书内切换到其他角色）
- 整个右栏可通过顶栏按钮折叠，腾出更多对话空间

### 路由结构

```
/                       → 重定向到 /worlds
/worlds                 → 世界书大厅（首页）
/worlds/create          → 创建世界书
/worlds/:id             → 世界书详情 + 角色列表
/worlds/:id/edit        → 编辑世界书
/worlds/:id/characters/create  → 创建角色
/characters/:id/edit    → 编辑角色
/chat/:conversationId   → 对话界面
/login                  → 登录
/register               → 注册
```

### 前端状态处理

```
加载状态：
  - 世界书/角色列表加载中 → 骨架屏（Skeleton），卡片轮廓闪烁
  - 对话消息加载中 → 三点跳动动画（LoadingDots）
  - SSE 流式连接中 → 闪烁光标 █

空状态：
  - 无世界书 → 插画 + "还没有世界书，创建你的第一个世界"
  - 世界书下无角色 → "这个世界还没有角色，添加一个吧"
  - 无会话历史 → "选择一个角色开始你的冒险"

错误状态：
  - LLM 调用失败 → 消息气泡内显示错误提示 + [重试] 按钮
  - SSE 断线 → 顶部 toast "连接中断，正在重连..."，自动重试 3 次
  - 网络异常 → 全局顶部横幅 "网络连接失败"
  - 表单提交失败 → 字段下方红色错误文字

对话页「+ 新对话」按钮：
  - 点击后弹出当前世界书的角色选择列表
  - 选中角色 → 创建新会话 → 进入对话
  - 也可选择"浏览其他世界"跳转回世界书大厅
```

## 五、核心数据模型

### 1. 用户 (User)

```
User
├── id
├── username          — 用户名（唯一）
├── email             — 邮箱（唯一）
├── password_hash     — 密码哈希
├── nickname          — 显示昵称
├── avatar            — 头像
├── created_at
└── updated_at
```

### 2. 世界书 (WorldBook)

```
WorldBook
├── id
├── name              — 世界名称（如「中土世界」「霍格沃茨魔法世界」）
├── description       — 简要介绍
├── setting           — 世界观设定（时代、地理、社会结构）
├── rules             — 世界规则（魔法体系、物理法则、禁忌）
├── lore              — 历史与传说
├── factions          — 势力/阵营（JSON 数组）
├── tags              — 分类标签（奇幻、科幻、现实、历史...）
├── cover_image       — 封面图路径
├── is_preset         — 是否为系统预设（预设不可删除）
├── created_by        — 创建者（外键 → User）
├── created_at
└── updated_at
```

### 3. 角色 (Character)

```
Character
├── id
├── world_id          — 所属世界书（外键）
├── name              — 角色名
├── avatar            — 头像路径
├── identity          — 身份描述（职业、地位、种族）
├── personality       — 性格特征（详细文本描述）
├── background        — 个人经历与故事
├── relationships     — 与其他角色的关系（JSON）
├── language_style    — 说话风格（口癖、句式、用词习惯）
├── knowledge         — 角色所知道的信息（不超出世界观范围）
├── greeting          — 开场白（首次对话时角色的第一句话）
├── tags              — 标签
├── created_at
└── updated_at
```

### 4. 会话 (Conversation)

```
Conversation
├── id
├── user_id           — 所属用户（外键 → User）
├── character_id      — 对话角色（外键 → Character）
├── world_id          — 所属世界书（外键 → WorldBook）
├── title             — 会话标题（自动生成或用户命名）
├── summary           — 对话摘要（定期更新）
├── message_count
├── created_at
└── updated_at
```

### 5. 消息 (Message)

```
Message
├── id
├── conversation_id   — 所属会话（外键 → Conversation）
├── role              — user / assistant
├── content           — 消息内容
├── token_count       — Token 消耗
└── created_at
```

### 图片存储方案

开发阶段采用本地文件存储，生产环境可切换为对象存储（如 MinIO / 阿里云 OSS）。

```
上传接口：POST /api/upload   → 返回文件路径
存储目录：/uploads/
├── covers/       ← 世界书封面
├── avatars/      ← 角色头像 + 用户头像
└── misc/         ← 其他

前端通过 /api/static/{path} 访问
```

### 权限模型

```
系统预设世界书/角色（is_preset=true）：
  - 所有用户可查看、可对话
  - 仅管理员可编辑/删除

用户创建的世界书/角色（is_preset=false）：
  - 创建者可编辑/删除
  - 其他用户可查看、可对话（公开）
  - 后期可扩展为私有/公开切换
```

## 六、Prompt 组装策略

每次调用 LLM 时，按以下结构组装 Prompt：

```
[System Prompt]
═══════════════════════════════════════
【世界观】
你所处的世界是「{world.name}」。
{world.setting}

【世界规则】
{world.rules}

【势力与阵营】
{world.factions}

【历史与传说】
{world.lore}

【你的身份】
你是 {character.name}，{character.identity}。

【你的性格】
{character.personality}

【你的经历】
{character.background}

【你的人际关系】
{character.relationships}

【说话风格】
{character.language_style}

【你知道的事情】
{character.knowledge}

【行为准则】
- 始终以 {character.name} 的身份回应，不要跳出角色
- 所有回答必须符合世界观设定，不能出现超出世界观的知识
- 保持角色的说话风格和性格特征
- 如果用户问的内容超出角色认知范围，以角色的方式表达困惑或不知
═══════════════════════════════════════

[长期记忆 — 语义检索的相关历史片段]
{retrieved_memories}

[短期记忆 — 最近 N 轮对话]
{recent_messages}

[User]
{user_input}
```

### Token 上下文管理

System Prompt（世界书 + 角色）+ 记忆 + 历史对话可能超出 LLM 上下文窗口，需分层截断：

```
总预算 = 模型上下文窗口（如 32K）

分配策略：
├── System Prompt（世界书 + 角色设定）  — 固定部分，最高优先级，预估 2K-4K
├── 回复预留                           — 预留 2K token 给模型回复
├── 长期记忆（语义检索）                — 最多 2K，超出则减少 Top-K 数量
├── 短期记忆（近期对话）                — 剩余空间全部分配
└── 若仍超限 → 裁剪最早的短期记忆轮次

实现：prompt_builder.py 中计算各部分 token 数，动态调整窗口大小
```

## 七、API 设计

```
# 世界书
GET    /api/worlds                         — 世界书列表（分页 + 标签筛选）
GET    /api/worlds/{id}                    — 世界书详情
POST   /api/worlds                         — 创建世界书
PUT    /api/worlds/{id}                    — 编辑世界书（仅创建者/管理员）
DELETE /api/worlds/{id}                    — 删除世界书（仅创建者/管理员）

# 角色
GET    /api/worlds/{world_id}/characters   — 某世界书下的角色列表
GET    /api/characters/{id}                — 角色详情
POST   /api/worlds/{world_id}/characters   — 在世界书下创建角色
PUT    /api/characters/{id}                — 编辑角色（仅创建者/管理员）
DELETE /api/characters/{id}                — 删除角色（仅创建者/管理员）

# 对话
POST   /api/chat/start                     — 开始新对话
         body: { character_id, world_id }
         返回: { conversation_id, greeting_message }
POST   /api/chat/{conversation_id}/send    — 发送消息（返回 SSE 流式响应）
         body: { content }
POST   /api/chat/{conversation_id}/stop    — 中断当前生成
GET    /api/chat/conversations             — 用户的会话列表（按 updated_at 降序）
GET    /api/chat/{conversation_id}         — 获取历史消息（分页）
         query: { page, page_size }
DELETE /api/chat/{conversation_id}         — 删除会话

# 文件上传
POST   /api/upload                         — 上传图片（封面/头像）
         body: multipart/form-data
         返回: { path }

# 用户
POST   /api/auth/register                  — 注册
POST   /api/auth/login                     — 登录（返回 JWT）
GET    /api/user/profile                   — 用户信息
PUT    /api/user/profile                   — 更新用户信息
```
